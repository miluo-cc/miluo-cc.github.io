Hugo部署记录

下载Hugo

  • 下载地址
  • 版本：v0.70.0
  • 平台：win10
  • 下载后解压是一个应用程序无需安装(不愧是go)，选好放置目录添加到环境变量
  • 验证是否安装成功：hugo version

创建网站

  • 使用命令：hugo new site [name]，生成根目录，目录结构如下：

    ├─archetypes
├─content
├─data
├─layouts
├─static
├─themes
└─config.toml

选择主题

  • 主题网站
  • 选择一个如mixedpaper下载到themes目录下：git clone https://themes.gohugo.io/mixedpaper/ themes/mixedpaper

创建一篇文章

  • 使用命令：hugo new post/test.md，生成在content目录下

  • 文件默认内容如下

    title: "test"
date: 2020-05-13T13:11:31+08:00
draft: true

  • 其中draft表示是否草稿文件，编辑完将其改为 false，否则编译时会跳过

测试

  • 使用命令：hugo server --theme=mixedpaper --buildDrafts
  • buildDrafts表示包括草稿
  • 浏览器打开 http://localhost:1313查看页面

修改配置文件

  • 修改config.toml，如下示例，具体配置要看对应主题的参数要求

    baseURL = "https://miluo-cc.github.io/"
languageCode = "zh-CN"
title = "miluo"
theme = "mixedpaper"
author = "miluo"
copyright = "&copy; 2020. All rights reserved."

部署到github

  • 使用命令：hugo --baseUrl="https://miluo-cc.github.io"

  • 项目根目录多出public文件夹，里面是整个网站

  • 初次部署

    $ cd public
$ git init
$ git remote add origin git@github.com:miluo-cc/miluo-cc.github.io.git
$ git add -A
$ git commit -m "first commit"
$ git push -u origin master

  • 访问https://miluo-cc.github.io查看效果

后续添加文件

  • hugo new post/new_file.md
  • hugo --buildDrafts
  • cd public
  • git add .
  • git commit -m "a new blog"
  • git push origin master
